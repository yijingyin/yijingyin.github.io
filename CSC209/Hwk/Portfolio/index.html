HW4: Synchronized Animations

View Hw4

Creates synchronized, dynamic animations with user controls.

Problem Solved:

  • Move multiple elements simultaneously in different diagonal directions across the screen
  • Allow for user control over animation speed
  • Start, stop, and reset animations on demand

Techniques Used:

  • Configurable Animation Objects: Structured configuration array with parameters for each circle's animation
  • Interval-Based Animation: Manual animation through setInterval() for precise control
  • Dynamic Element Positioning: Direct CSS property manipulation for animation effects
  • Animation Management: Functions to start, track, and clear animations
  • User-Controlled Speed: Slider input element for adjusting animation rate

HW5: Vector Visualization

View Hw5

Visualizes vectors and their movement in a 2D space.

Problem Solved:

  • Creating a dynamic, interactive environment for vector behavior demonstration
  • Showing how multiple vectors move and bounce within boundaries
  • Visualizing both position and direction of vectors simultaneously
  • Allowing users to trace vector movement paths over time
  • Providing controls to adjust vector count, toggle animation, and reset the simulation

Techniques Used:

  • Canvas-based Animation: HTML Canvas with requestAnimationFrame() for smooth performance
  • Vector Mathematics: Implementation of vector creation, normalization, and movement calculations
  • Collision Detection and Response: Boundary collision detection with appropriate vector reflection
  • Path Tracing: Array of previous positions for drawing historical paths
  • Parameterized Simulation: User controls for particle count and animation speed

HW6: Conway's Game of Life

View Hw6

Implementation of Conway's Game of Life cellular automaton.

Problem Solved:

  • Implementing core rules for cell life, death, and reproduction
  • Providing visual grid representation of the cellular automaton
  • Enabling user interaction to toggle cells manually
  • Allowing animation control (start, stop, step)
  • Supporting different canvas sizes for varying complexity
  • Offering random cell initialization for spontaneous pattern generation

Techniques Used:

  • Canvas-based Rendering: HTML Canvas for efficient visualization
  • Two-dimensional Array Data Structure: 2D array for grid state representation
  • Neighbor Counting Algorithm: Function to calculate living neighbors for each cell
  • State Management: Two-phase update approach to prevent update interference
  • Event-driven Interaction: Event listeners for user interaction and simulation control
  • Timed Animation Loop: requestAnimationFrame with setTimeout for controlled generation timing
  • Responsive Configuration: Adjustable canvas size and random cell initialization
